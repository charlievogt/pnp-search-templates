{{!-- Trailing-12-months tab strip + card grid for PnP Modern Search. The tab strip is generated entirely client-side from "now" — no PnP Verticals, no per-tab Search Results web part, no SPFx helper additions, no monthly rebuild. The page works for years without ever being edited. Two tricks make it portable: 1. JSONparse a static array of duration strings ["0 month", "1 month", ..., "11 month"]. Iterating that array gives us per-month context without an out-of-template loop or build step. 2. Nested moment + getDate: {{getDate (moment (moment) subtract=dur) 'YYYY-MM'}} Inner `(moment)` produces today as a date string; middle `(moment STRING subtract=dur)` shifts it; outer `getDate` formats. The chain works on both PnP 4.19.1 (helper-date) and 4.21+ (the rewritten helperMomentCompat), so the template is portable across versions. Tab switching is pure CSS via hidden + :checked ~ sibling selectors — no JavaScript. Active-tab styling uses :has() because the labels live in a different DOM container than the radios. Expected `data.items` shape: Title — display name (required) EffectiveDate — date string parseable by dayjs (required for bucketing) Position — optional, shown in card meta Location — optional, shown in card meta Narrative — optional, HTML, shown as body PhotoUrl — optional, shown as card image Wire any data source (SharePoint Search, Microsoft Search, or a custom extensibility library) that returns these fields. See the project README for example web-part configurations. --}}
